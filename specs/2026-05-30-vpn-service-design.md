# VPN 服务（TikTok / Twitter / WhatsApp 海外运营）设计文档

> 创建时间：2026-05-30
> 业务场景：为国内做 TikTok / Twitter(X) / WhatsApp / Facebook 海外运营的客户提供稳定、IP 纯净度高的翻墙网络服务
> 商业模式：自营订阅服务（月付 / 年付），独立收款，远程协助客户装机
> 文档作者：与 Claude 协作整理

---

## 1. 业务背景与目标

### 1.1 客户画像
- 国内（中国大陆）做海外社媒运营的个人或工作室
- 业务以 TikTok、Twitter(X)、WhatsApp、Facebook 等为主
- 客户规模预期：6–12 个月内 10–50 个客户，100–500 台设备
- 客户对**稳定性**与**IP 纯净度**要求极高（直接关系账号存活）

### 1.2 核心目标
1. 客户在中国大陆能稳定访问 TikTok / Twitter / WhatsApp 等主流境外服务
2. 尽可能降低账号被风控 / 被封号风险
3. 控制单客户成本，保证利润空间
4. 客户可按月或按年订阅，订阅链接可配置有效期与终端数

### 1.3 非目标
- 不做客户自助注册 / 自助下单（管理员手动开通）
- 不集成在线支付（线下收款）
- 不为单客户提供多 IP 切换 / 多地区切换（每客户专属 1 IP）
- 不做爬虫 / 数据采集类用途

## 2. 总体架构

### 2.1 数据流图

```
┌────────────────────────────────────────────────────────────────┐
│  中国客户终端 - 单客户 ≤10 台设备                                │
│  iOS:Shadowrocket / Android:NekoBox / Win:v2rayN / Mac:ClashX  │
└────────────────────────┬───────────────────────────────────────┘
                         │
                         │ ① VLESS+Reality（主） / Trojan+TLS（备）
                         │   抗墙加密通道，穿透 GFW
                         ▼
┌────────────────────────────────────────────────────────────────┐
│  该客户专属海外 VPS（美/日/新/港 任一地区）                       │
│  - Xray-core 内核                                               │
│  - 3x-ui 面板（订阅链接 / 过期 / 设备数限制）                     │
│  - inbound：Reality + Trojan                                    │
│  - outbound：SOCKS5 → 该客户专属住宅 IP                          │
└────────────────────────┬───────────────────────────────────────┘
                         │
                         │ ② SOCKS5 转发
                         ▼
┌────────────────────────────────────────────────────────────────┐
│  该客户专属住宅 IP（IPRoyal Static ISP / 922S5 静态独享）        │
│  与 VPS 同地区配套                                               │
└────────────────────────┬───────────────────────────────────────┘
                         │
                         │ ③ 从住宅 IP 出公网
                         ▼
            TikTok / Twitter(X) / WhatsApp / Facebook
            （平台视角：一户家庭住宅 + 多台设备登录，符合正常画像）
```

### 2.2 核心设计原则

| 原则 | 含义 | 价值 |
|---|---|---|
| **一客一节点** | 1 客户 = 1 VPS + 1 住宅 IP | 故障域物理隔离；某客户被风控不影响他人；IP 纯净度由本客户使用方式决定 |
| **VPS 与住宅 IP 同地区** | 美客户 → 美 VPS + 美住宅 IP | 出口地区一致，降低风控触发 |
| **多协议冗余** | Reality + Trojan 双协议 | 某协议被识别 / 限速时客户端自动切换 |
| **统一订阅** | 一条 URL 包含多协议节点 | 跨平台客户端同一订阅，运维简单 |
| **按需采购** | VPS 与 IP 接单后再开 | 不囤资源，零空置成本 |

## 3. 技术栈

| 层级 | 选型 | 备选 / 升级路径 |
|---|---|---|
| 客户端（iOS） | Shadowrocket（付费 $2.99，需海外 Apple ID） | Stash / Quantumult X |
| 客户端（Android） | NekoBox / v2rayNG | Clash for Android |
| 客户端（Windows） | v2rayN / Nekoray | Clash Verge |
| 客户端（Mac） | ClashX Pro / Stash | NekoRay |
| 抗墙协议 | **VLESS + Reality**（主） | Trojan + TLS（备） / Hysteria2（极端环境） |
| 代理核心 | Xray-core | Sing-box（未来可替） |
| 后台面板（起步） | **3x-ui** | 客户≥15 时迁 Xboard 总控 |
| 后台 HTTPS | Caddy（自动 Let's Encrypt） | Nginx + acme.sh |
| 住宅 IP 接入方式 | Xray outbound → SOCKS5 | HTTP 代理（不推荐） |
| 客户登记 | Notion / 飞书表格 | 后期可上轻量 CRM |

### 3.1 为什么选 Reality 而不是传统 VMess+WS+TLS

| 维度 | VLESS+Reality | VMess+WS+TLS |
|---|---|---|
| 抗墙能力 | ⭐⭐⭐⭐⭐（当前最强） | ⭐⭐⭐ |
| 是否需买域名 | ❌ 不需要 | ✅ 必须 |
| 是否需 SSL 证书 | ❌ 不需要 | ✅ 必须 |
| 部署复杂度 | 低 | 中 |
| 客户端兼容 | Shadowrocket / v2rayN / NekoBox / ClashX 均支持 | 同上 |

**结论：Reality 既省域名钱，又抗墙最强，零理由不用。**

### 3.2 为什么起步用 3x-ui 而不是 Xboard

- 你的模式是「1 客户独占 1 节点」，Xboard 的「多用户共享节点 + 流量计费」对你完全是浪费
- 3x-ui 单机面板原生支持：**过期时间 + 客户端数量限制 + 流量限额**
- 客户≥15 个时再迁移到 Xboard 总控（迁移成本不高）

- Vultr 强烈推荐：可按小时计费（每月封顶 = 月费），用完 destroy 不浪费

### 4.2 住宅 IP 供应商推荐

| 厂商 | 产品类型 | 单 IP 月费 | 付款 | 推荐度 |
|---|---|---|---|---|
| **IPRoyal Royal Residential（静态 ISP）** | 静态住宅 IP，独享 | $2.40–7/IP | PayPal | 🥇 主力 |
| **922S5 静态独享** | 静态住宅 IP | ¥30–60/IP | 支付宝 / 微信 | 🥈 备用 |
| **NodeMaven** | 高纯净静态 ISP | $8–12/IP | PayPal / 卡 | 🥉 高端线 |
| **BrightData / Oxylabs** | 企业级 | $15+/IP | 企业付款 | 暂不考虑（贵） |

### 4.3 关键概念：必须买"静态住宅 ISP"，不要流量型

| 类型 | 计费 | 适合场景 | 是否买 |
|---|---|---|---|
| 流量型住宅代理（动态 IP 池） | 按 GB（$3–15/GB） | 爬虫、SEO 验证 | ❌ 绝不 |
| **静态住宅 ISP（独占 IP）** | 按 IP × 月（$2.4–12/IP/月） | 账号养号、长期挂机 | ✅ 这是你要的 |

**绝不能买流量型的两个理由：**
1. 10 台设备日跑 10–50GB，按流量算月成本上千美金，赔本
2. 流量型动态换 IP，账号会因「频繁登录地变化」被封号 — 与"稳定+纯净"卖点直接冲突

### 4.4 IP 纯净度自检三件套（每个新 IP 上机前必测）

| 工具 | 检查项 | 合格标准 |
|---|---|---|
| https://whoer.net | 综合匿名分 | 80%+ |
| https://scamalytics.com | 风险评分 | Low Risk |
| https://ipinfo.io | IP 类型 | 必须是 `residential`，不能是 `datacenter` / `hosting` |

## 5. 后台管理需求

### 5.1 订阅链接配置项（每个客户一条）

| 字段 | 含义 | 示例 |
|---|---|---|
| 客户标识 | 内部备注名 | `2026-客户A` |
| 有效期 | 订阅到期时间 | 3 个月 / 1 年 |
| 终端数上限 | 最大同时连接设备数 | 10 |
| 流量上限 | 月流量配额（可选） | 不限 / 1TB |
| 协议 | 启用的入站协议 | Reality + Trojan |

3x-ui 面板原生支持以上全部字段，无需二次开发。

### 5.2 管理员后台职能

- 创建 / 续期 / 暂停 / 删除客户
- 复制订阅链接发给客户
- 实时查看在线设备数、流量使用
- 客户到期前通过表格 / 提醒通知续费（手动）

## 8. iOS 装机注意事项（Shadowrocket）

- Shadowrocket 是 iOS 收费 App（约 $2.99）
- **必须用海外（美区 / 日区 / 港区）Apple ID 才能下载**
- 国内客户初次装机建议流程：
  1. 你提前注册好的美区 Apple ID 借给客户登录购买（或客户自己注册）
  2. 在 App Store 切美区 → 搜索 Shadowrocket → 购买（用礼品卡或支持 PayPal）
  3. 装好后切回客户原 Apple ID（保留 Shadowrocket 在已购列表）
  4. 远程导入订阅链接，测试访问

可考虑做一份「Shadowrocket 购买 + 安装图文教程」给客户自助。

## 9. 演进路径（10 → 50 客户）

| 阶段 | 客户数 | 架构 | 运维强度 |
|---|---|---|---|
| **起步期** | 1–15 | 单 VPS + 单 3x-ui，每客户独立面板 | 手动 SOP，30–45 分钟/客户 |
| **规模化** | 15–50 | 总控 Xboard + 多节点同步，自动化部署脚本 | 脚本化部署，10–15 分钟/客户 |
| **企业化** | 50+ | 多区域多面板 + 客户自助续费门户 + 收款集成 | 团队化运维 |

每跨一个阶段前重新评估：是否需要专职运维 / 是否上自助门户 / 是否引入合伙人。

## 10. 风险与对策

| 风险 | 概率 | 对策 |
|---|---|---|
| GFW 封锁 Reality 协议 | 低 | 备 Trojan 协议；极端时切 Hysteria2 |
| VPS IP 被墙 | 中 | Vultr 可换 IP（$5）；备用 BWG 等冷备 |
| 住宅 IP 突然变脏 | 中 | 接入前三件套测；每月抽测；变脏立即换 IP |
| 客户违规使用导致 IP 风险 | 中 | 用户协议禁止违规用途；保留切断权 |
| 上游住宅 IP 厂跑路 | 低 | 同时维护 IPRoyal + 922S5 双供应商 |
| 客户对接麻烦（Apple ID、远程协助） | 高 | 做好图文教程；远程时段集中安排 |
| 收款合规 | 中 | 个人收款，控制金额，必要时注册个体户 |
